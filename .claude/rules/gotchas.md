## Gotchas

- `middleware.ts` is named `proxy.ts` ‚Äî exports `proxy` function, not `middleware`
- Dev server is usually running in another terminal ‚Äî check `lsof -iTCP:3000` before starting `pnpm dev`
- `pnpm fix` runs the full fix chain via Turbo dependency graph (`fix:css ‚Üí fix:md ‚Üí fix:ultracite`)
- `build:ci` skips env validation, OpenAPI gen, and sitemap ‚Äî use `pnpm build` for local verification
- `typescript.ignoreBuildErrors: true` in next.config ‚Äî TS errors do NOT fail production builds
- `reactStrictMode` is disabled (commented out)
- No test suite ‚Äî `pnpm test` exits 0 with "no test specified". Cypress installed but no specs
- CI runs lint checks only (no build gate) ‚Äî build failures surface on Vercel
- `pnpm lint:ultracite` requires Next.js generated types ‚Äî CI runs `pnpm next:typegen` first. Locally, types already exist if `next dev` or `pnpm build` has run; only run `pnpm next:typegen` manually if lint fails on missing generated types
- `prebuild:next` generates OpenAPI spec before every `next build`
- Custom image loader: `src/utils/cloudflareImageLoader.ts` (not Next.js default optimizer)
- Sentry tunnel: events proxied through `/skynet` to bypass ad-blockers
- `axios` is in dependencies but the rule is `fetch` only ‚Äî legacy, do not use for new code
- Node engine: `^22.14.0`
- cspell words are inline in `cspell.json` ‚Äî add words manually only when CI flags a legitimate term
- Commit body lines must be under 100 characters (`commitlint` enforces `body-max-line-length`)
- `profiles.category_order` updates: use batch concatenation (`|| v_new_category_ids`), NOT read-modify-write loops ‚Äî the Edge Function processes queue messages in parallel (`Promise.allSettled`), and a for-loop that SELECTs the full array into a variable then UPDATEs will cause lost writes. See `20260209` migration for the correct pattern.
- `src/pages/[category_id].tsx` catches ALL single-segment paths ‚Äî new App Router pages at `/foo` will 404 in dev because Pages Router dynamic routes take precedence
- New public pages must be added to `PUBLIC_PATHS` in `src/utils/constants.ts` ‚Äî otherwise `proxy.ts` middleware treats them as auth-protected
- `.agents/` and `.claude/` are excluded from all linters/formatters (oxlint, oxfmt, cspell, markdownlint) ‚Äî but NOT gitignored as directories
- `AGENTS.md` is a symlink to `CLAUDE.md` ‚Äî do not replace with a regular file
- `next.config.ts` has experimental flags (`prefetchInlining`, `appNewScrollHandler`, `sri`) ‚Äî check Next.js docs before adding/removing
- Error boundaries (`error.tsx`, `global-error.tsx`) use `unstable_retry()` from `next/error` (not `reset()`) ‚Äî re-fetches RSC data on retry
- Sentry tree-shaking (`bundleSizeOptimizations`, `webpack.treeshake`) is webpack-only ‚Äî no-op for Turbopack builds. SDK v10 has no Turbopack tree-shaking support yet
- `next build` defaults to Turbopack in v16 ‚Äî hard-fails (`process.exit(1)`) if config has `webpack` key without `turbopack` key
- `@next/bundle-analyzer` is Turbopack-safe ‚Äî detects `TURBOPACK` env var and skips (no webpack key added)
- Sentry `disableLogger` is deprecated ‚Äî use `webpack.treeshake.removeDebugLogging`; top-level `reactComponentAnnotation` ‚Üí `webpack.reactComponentAnnotation`
- `/discover` page is blank without JavaScript ‚Äî `[category_id].tsx` uses `useMounted()` which gates all rendering behind client-side hydration. `getServerSideProps` fetches data but the component won't render it server-side. Search engines see an empty page
- `npx skills add <pkg> -a "Claude Code" -y` ‚Äî installs to `.agents/` (universal) + `.claude/` only. Without `-a`, `-y` installs to ALL agents creating unwanted dirs
- `skills-lock.json` at project root is auto-generated by `npx skills` ‚Äî commit it (used by `npx skills experimental_install`)
- `pnpm release:dryrun` is interactive (prompts for version) ‚Äî can't run via Bash tool. Leaves `package.json` version bump; clean up with `git checkout -- package.json`
- CHANGELOG.md is excluded from all linters/formatters (oxfmt, markdownlint, cspell) ‚Äî formatting comes directly from Handlebars templates in `scripts/release-it/templates/`
- Notable Changes in changelog = `feat`/`fix` commits with a body; Other Notable Changes = `refactor`/`perf`/`docs` with a body; Breaking Changes = `!` in header
- GitHub blockquotes (`>`) render with constrained max-width ‚Äî don't use for full-width content in CHANGELOG.md or GitHub releases
- `@release-it/conventional-changelog` pinned to v9.x ‚Äî v10.x has `whatBump` bug (unwaited `loadPreset`)
- For targeted formatting, run `npx oxfmt <file>` directly
- Shell scripts must be bash 3.2 compatible (macOS default) ‚Äî no `declare -A`, use `case` + temp files
- Release PRs use `release` label ‚Äî skips CodeRabbit review
- Release pipeline: `pnpm release:pr:yes` ‚Üí `gh pr merge --merge --admin` ‚Üí CI runs release-it ‚Üí CI backmerges main‚Üídev (clears `docs/API_CHANGELOG.md`) ‚Üí verify tag + GitHub Release. Full automation via `/release` skill
- `pnpm release` requires `GITHUB_TOKEN` env var ‚Äî the changelog writer's `getGithubCommits()` fetches commit author data from GitHub API
- `release-pr.sh` detects existing release PRs and offers to delete+recreate ‚Äî no need to manually clean up before re-running
- `release-pr.sh --yes` / `-y` flag auto-confirms all prompts ‚Äî use for agent-native / CI execution
- `gh pr merge` on `main` requires `--admin` ‚Äî branch protection blocks direct merge even with `release` label
- Backmerge verification: `git log origin/dev..origin/main` should be empty after successful release ‚Äî if it shows the release commit, the backmerge didn't preserve the merge commit SHA (previously caused by `--amend` in API changelog clear, fixed in `81b92e78`)
- `docs/API_CHANGELOG.md` is auto-appended by CI on each push to `dev`, posted as PR comment during release, and cleared during backmerge
- `.ncurc.cjs` pins packages that can't be upgraded (mirrors `.github/renovate.json` blocks) ‚Äî keep both in sync
- GitHub Actions use pinned commit SHAs with version comments ‚Äî use `gh api repos/{owner}/{repo}/git/ref/tags/{tag}` to get SHAs when upgrading
- CI cspell may flag unknown words ‚Äî hyphen-split words (e.g. "app-svgs" ‚Üí "svgs") may need manual additions to `cspell.json` `words` array
- `lint-staged` glob must be `*.{js,jsx,ts,tsx,cjs,mjs,json,jsonc}` not `*` ‚Äî ultracite crashes on non-JS files like `.txt`
- `oxlint-disable-next-line` doesn't work for JSX props on different lines ‚Äî use block-level `/* oxlint-disable rule */` instead
- Comment directive split: native oxlint rules use `oxlint-disable`, jsPlugin rules (`@tanstack/query/*`, `regexp/*`, `perfectionist/*`, `react-x/*`) keep `eslint-disable`
- `promise-function-async` is off ‚Äî it auto-adds `async` to Promise-returning functions, but the rule is: only add `async` when `await` is present in the function body
- oxlint `--deny RULE` CLI flag does NOT override config-level `"off"` ‚Äî to test a disabled rule, copy `.oxlintrc.json` to a temp file, edit the copy, run `npx oxlint -c <copy>` from project root (relative `extends` paths require it), then `trash` the copy
- `oxlint-disable-next-line` must be on the line immediately before the violation ‚Äî for multi-line expressions, place on the line with the violation, not the wrapping expression above
- `typeAware: true` and `typeCheck: true` are enabled ‚Äî all type-aware rules active globally. `no-unsafe-*` + `no-non-null-assertion` disabled in legacy folders via `.oxlintrc.json` overrides (`src/pages/api/`, `src/async/`, `src/pageComponents/`, `src/utils/{worker,file-upload,helpers,apiHelpers}.ts`, `scripts/`). New code is fully enforced
- `database-generated.types.ts` is in oxlint `ignorePatterns` ‚Äî Supabase CLI generates types that trigger `no-redundant-type-constituents`
- `cypress/e2e/**` and `cypress/support/**` are in oxlint `ignorePatterns` ‚Äî `Cypress.env()` is falsely flagged as deprecated
- `prefer-nullish-coalescing` auto-fix converts `||` to `??` ‚Äî use block-level `/* oxlint-disable prefer-nullish-coalescing */` when `||` is intentional (boolean conditions, empty-string-as-falsy chains)
- `no-misused-promises` / `no-floating-promises`: async callbacks in `onSubmit`, `onClick`, `startTransition`, `onValueChange` must wrap: `() => { void (async () => { ... })(); }`. Form `onSubmit` specifically: `(e) => { e.preventDefault(); void handleSubmit(fn)(); }`
- `createServerServiceClient()` (`@/lib/supabase/service`) is synchronous ‚Äî don't `await` it
- `require-await` checks the OUTER function scope only ‚Äî `await` inside nested callbacks (`startTransition(async () => { await ... })`) doesn't count. Remove `async` from the outer function, keep it on the inner callback
- `npx oxlint <changed-files>` only checks listed files ‚Äî CI runs `pnpm lint:ultracite` on ALL files. When enabling new rules, run full `pnpm lint:ultracite` locally before pushing
- `perfectionist/sort-objects` + `sort-union-types` auto-fix oscillates with oxfmt on multi-line objects ‚Äî upstream wontfix (oxc#20210). To enable: fix all violations manually first; auto-fix is stable only on already-sorted code
